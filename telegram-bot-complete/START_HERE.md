## 🚀 БЫСТРЫЙ СТАРТ (3 шага)

### Шаг 1: Распакуйте архив

```bash
unzip telegram-bot-complete.zip
cd telegram-bot-complete
```

### Шаг 2: Настройте окружение

```bash
# Установите зависимости
pip install -r requirements.txt

# Создайте .env файл
cp .env.example .env
nano .env  # или любой редактор
```

В `.env` укажите:
```env
BOT_TOKEN=ваш_токен_от_BotFather
ADMIN_ID=ваш_telegram_id
DATABASE_PATH=bot.db
AI_API_KEY=
```

### Шаг 3: Запустите

```bash
python -m app.main
```

**Готово!** Найдите бота в Telegram и отправьте `/start` 🎉

## 🔧 Настройка

### Изменить RSS источники:
Откройте `app/config.py` → секция `RSS_SOURCES`

### Изменить ключевые слова:
Откройте `app/config.py` → секция `ASSET_KEYWORDS`

### Изменить расписание:
Откройте `app/config.py`:
```python
INSTANT_CHECK_INTERVAL = 15  # минут
DAILY_NOTIFICATION_TIME = "09:00"
WEEKLY_NOTIFICATION_DAY = "mon"  # mon, tue, wed...
```

### Добавить новые активы:
```python
AVAILABLE_ASSETS = {
    "crypto": ["BTC", "ETH", "SOL"],  # добавили SOL
    ...
}

ASSET_KEYWORDS = {
    "SOL": ["solana", "sol"],  # ключевые слова
    ...
}
```

## 🆘 Частые проблемы

### 1. ModuleNotFoundError: No module named 'app'

**Решение:**
```bash
cd путь/к/telegram-bot-complete
python -m app.main  # НЕ python app/main.py
```

### 2. ValueError: Invalid weekday name "monday"

**Решение:** См. файл `FIX_SCHEDULER.md`

Или измените в `app/config.py`:
```python
WEEKLY_NOTIFICATION_DAY = "mon"  # было "monday"
```

### 3. Bot token is invalid

**Проверьте:**
- Токен в `.env` правильный
- Нет лишних пробелов
- Формат: `BOT_TOKEN=123456789:ABCdefGHI...`

### 4. База данных заблокирована

Убедитесь что запущена только одна копия бота:
```bash
ps aux | grep "app.main"
```
