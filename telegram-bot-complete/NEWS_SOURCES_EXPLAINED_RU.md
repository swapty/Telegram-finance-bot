# Источники новостей и фильтрация - подробное объяснение

## 📰 Откуда бот берет новости?

Бот использует **RSS-ленты** (публичные новостные потоки) от известных финансовых сайтов.

### 🔗 Текущие источники:

#### 💰 Криптовалюты (Crypto):
1. **CoinTelegraph** - https://cointelegraph.com/rss
   - Один из крупнейших крипто-новостных сайтов
   - Новости о Bitcoin, Ethereum, альткоинах
   - Обновляется каждые несколько минут

2. **CoinDesk** - https://coindesk.com/arc/outboundfeeds/rss/
   - Авторитетный источник крипто-новостей
   - Аналитика, цены, регуляция
   - Профессиональная журналистика

#### 📈 Акции (Stocks):
1. **Yahoo Finance** - https://feeds.finance.yahoo.com/rss/2.0/headline
   - Мировые финансовые новости
   - Новости компаний, рынков
   - Обновляется постоянно

2. **CNBC** - https://www.cnbc.com/id/100003114/device/rss/rss.html
   - Деловые новости США
   - Фондовый рынок, экономика
   - Профессиональный контент

#### 🥇 Металлы (Metals):
1. **Kitco** - https://www.kitco.com/rss/metals.xml
   - Специализация на драгоценных металлах
   - Золото, серебро, платина
   - Цены и аналитика

## 🤖 Используется ли AI?

### Текущая версия (MVP):
**НЕТ, AI не используется** 

Фильтрация работает через **простой поиск ключевых слов**:

```python
# Пример фильтрации
Если пользователь выбрал BTC:
  Ищем в новости слова: "bitcoin", "btc"
  Если найдено → отправляем новость
```

### Преимущества текущего подхода:
- ✅ Быстро работает
- ✅ Не требует API ключей
- ✅ Бесплатно
- ✅ Надежно

### Недостатки:
- ❌ Может пропустить релевантные новости
- ❌ Может отправить нерелевантные
- ❌ Нет оценки важности новости

## 🧠 Как можно добавить AI? (Будущее)

### Вариант 1: OpenAI GPT (рекомендуется)

**Что можно улучшить:**
- Оценка важности новости (1-10)
- Определение тональности (позитив/негатив)
- Категоризация (регуляция/цена/технологии)
- Генерация краткого саммари

**Пример использования:**
```python
import openai

# Для каждой новости
response = openai.ChatCompletion.create(
    model="gpt-4o-mini",  # Дешевая модель
    messages=[{
        "role": "user",
        "content": f"""
        Оцени важность этой новости для инвестора (1-10):
        {news_title}
        
        Верни только число.
        """
    }]
)

importance = int(response.choices[0].message.content)
if importance >= 7:
    send_to_user(news)
```

**Стоимость:**
- GPT-4o-mini: ~$0.15 за 1000 запросов
- Для 100 пользователей × 10 новостей/день = $0.15/день
- ~$4.5/месяц

### Вариант 2: Claude API (Anthropic)

**Преимущества:**
- Лучше понимает контекст
- Можно дать детальные инструкции

**Стоимость:**
- Claude Haiku: $0.25 за 1М токенов
- Еще дешевле чем GPT

### Вариант 3: Бесплатные модели (Hugging Face)

**Модели:**
- DistilBERT для классификации
- Sentiment analysis модели
- Запускать локально

**Преимущества:**
- ✅ Бесплатно
- ✅ Работает offline

**Недостатки:**
- ❌ Нужен GPU или долго работает
- ❌ Хуже качество

## 🎯 Как работает фильтрация сейчас?

### Шаг 1: Получение новостей
```python
# Каждые 15 минут (для instant)
all_news = fetch_rss_feeds()
# Получаем ~50 новостей из всех источников
```

### Шаг 2: Фильтрация по ключевым словам
```python
# Пользователь выбрал: BTC, ETH
user_assets = ["BTC", "ETH"]

# Ключевые слова
ASSET_KEYWORDS = {
    "BTC": ["bitcoin", "btc"],
    "ETH": ["ethereum", "eth", "ether"]
}

# Проверяем каждую новость
for news in all_news:
    title_and_text = news.title + news.summary
    
    if "bitcoin" in title_and_text.lower():
        send_to_user(news)  # Отправляем!
```

### Шаг 3: Проверка дубликатов
```python
# Создаем hash новости
news_hash = md5(title + link)

# Проверяем в базе
if not already_sent(user_id, news_hash):
    send_news(user_id, news)
    mark_as_sent(user_id, news_hash)
```

## 🔧 Как добавить больше источников?

Откройте `app/config.py` и добавьте:

```python
RSS_SOURCES = {
    "crypto": [
        "https://cointelegraph.com/rss",
        "https://coindesk.com/arc/outboundfeeds/rss/",
        # Новые источники:
        "https://cryptonews.com/news/feed/",
        "https://bitcoinmagazine.com/.rss/full/",
    ],
    "stocks": [
        "https://feeds.finance.yahoo.com/rss/2.0/headline",
        "https://www.cnbc.com/id/100003114/device/rss/rss.html",
        # Новые источники:
        "https://www.bloomberg.com/feed/podcast/markets",
        "https://www.marketwatch.com/rss/",
    ],
    # Добавить новый рынок:
    "forex": [
        "https://www.forexlive.com/feed/news",
    ],
}
```

### Как найти RSS ленты?

1. Откройте сайт новостей
2. Найдите иконку RSS (обычно в футере)
3. Или добавьте `/rss` или `/feed` к URL
4. Проверьте в браузере - должен открыться XML

## ❓ Часто задаваемые вопросы

### Q: Можно ли добавить свои источники?
**A:** Да! Просто добавьте RSS URL в `config.py`

### Q: Почему не используется AI сейчас?
**A:** Чтобы сделать MVP быстрее и бесплатнее. AI добавим когда будут пользователи.

### Q: Можно ли добавить Telegram каналы?
**A:** Да, но это сложнее. Нужен отдельный парсер Telegram.

### Q: Какие еще рынки можно добавить?
**A:** Forex, commodities (нефть, газ), недвижимость, экономические индикаторы.

### Q: Как часто обновляются RSS ленты?
**A:** Обычно каждые 5-30 минут на стороне источника.

### Q: Сколько новостей в день получит пользователь?
**A:** Зависит от активов. В среднем 5-15 новостей/день на 1 актив.
